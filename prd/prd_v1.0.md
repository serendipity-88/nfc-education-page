## 2. 需求背景

### 2.1 现状问题
- 支付宝"碰一下"支付服务推广中，大量用户未开启 NFC 功能
- 用户对 NFC 存在两大核心顾虑：
  1. **安全顾虑**：担心被远程盗刷、不安全
  2. **耗电顾虑**：担心常开 NFC 会消耗电量

### 2.2 现有方案问题
- 纯图文静态页面，缺乏互动体验
- 用户难以真正理解 NFC 技术原理
- 转化率低下

---

## 3. 需求目标

| 目标类型 | 指标 | 目标值 |
|----------|------|--------|
| 业务目标 | NFC 功能打开率 | 提升 30% |
| 用户目标 | 消除安全顾虑 | 70% 用户理解 NFC 安全距离 |
| 用户目标 | 消除耗电顾虑 | 80% 用户理解 NFC 几乎不费电 |

---

## 4. 用户与使用场景

### 4.1 目标用户
- 支付宝内看到"碰一下"服务但未开启 NFC 的用户
- 年龄层：25-65 岁，包含大量中老年用户
- 特征：对新技术有顾虑，需要简单易懂的教育

### 4.2 使用场景
| 场景 | 用户心理 | 设计方案 |
|------|----------|----------|
| 支付时被提示开启 NFC | 犹豫、担心安全 | 安全感教育页首屏强调"4cm 安全距离" |
| 想开启但担心费电 | 耗电焦虑 | 电量教育页用互动实验证明"几乎不费电" |
| 频繁开关 NFC | 以为能省电 | 实验 3 展示"频繁开关省的电微乎其微" |

## 6. 详细方案

### 6.1 安全感教育页

#### 6.1.1 首屏布局

```
┌─────────────────────────────┐
│   🔋 NFC 波纹动画            │
│                             │
│  无需开通                    │
│  碰一下就支付                │
│                             │
│  不用打开支付宝 快人一步      │
│  ✓ 安全  ✓ 成熟              │
└─────────────────────────────┘
```

#### 6.1.2 安全距离滑块
- **互动逻辑**：拖动滑块，显示不同距离下的 NFC 通信状态
- **状态映射**：
  - 0-4cm: ✅ 安全通信中（绿色）
  - 4-6cm: ⚠️ 信号不稳定（黄色）
  - 6cm+: ❌ 信号中断（红色）

#### 6.1.3 技术成熟度数据
| 场景 | 数据 |
|------|------|
| 公交卡 | 中国 338 城支持，每天 1.2 亿人使用 |
| 智能手机 | 2023 年新手机 100% 支持 NFC |
| 门禁卡/酒店卡 | 全国 90%+ 酒店在用 |

### 6.2 电量教育页

#### 6.2.1 实验 1：碰一下支付
- **互动方式**：长按按钮模拟连续"碰一下"
- **数据逻辑**：
  - 每次碰一下耗电：0.001%
  - 长按时每 100ms 计一次
- **显示内容**：
  - 电池进度条实时更新
  - 已模拟次数
  - 总耗电百分比

#### 6.2.2 实验 2：时长选择器
- **互动方式**：点击时长按钮 (1h/2h/4h/8h/12h/24h)
- **数据逻辑**：
  | 时长 | 耗电量 | 文案 |
  |------|--------|------|
  | 1 小时 | 0.001% | 喝杯咖啡的功夫 |
  | 2 小时 | 0.002% | 看一部电影的时间 |
  | 4 小时 | 0.003% | 短途旅行的车程 |
  | 8 小时 | 0.007% | 睡一觉起来 |
  | 12 小时 | 0.01% | 半个白天 |
  | 24 小时 | 0.02% | 整整一天 |

#### 6.2.3 实验 3：折腾这么久，省了多少？
- **互动方式**：选择每天开关次数 (1/3/5/10/20/自定义)
- **核心逻辑**：用生活化场景展示"频繁开关省的电微乎其微"
- **数据计算**：
  - 开关一次耗电：0.0001%
  - 常开一天耗电：0.02%
  - 每月常开耗电：0.02% × 30 = 0.6%
  - 每月省电 = 常开 30 天耗电 - (每天开关次数 × 30 × 每次耗电)

- **生活化场景换算率**：
  | 场景 | 每秒耗电 | 分钟耗电 |
  |------|----------|----------|
  | 刷短视频 | 0.005% | 0.3% |
  | 打电话 | 0.0025% | 0.15% |
  | 听音乐 | 0.0017% | 0.1% |
  | 电视剧 | 0.3%/集 | - |
  | 导航 | 0.15%/分钟 | - |

- **文案映射**：
  | 每天次数 | 每月省电 | 场景文案 |
  |---------|---------|---------|
  | 1 次 | 0.597% | 📱 这点电，只够刷 2 分钟短视频 |
  | 3 次 | 0.591% | 📞 这点电，只够打 4 分钟电话 |
  | 5 次 | 0.585% | 🎵 这点电，只够听 1 首歌 |
  | 10 次 | 0.57% | 📺 这点电，只够看半集电视剧 |
  | 20 次 | 0.54% | 🗺️ 这点电，只够导航 3.5 分钟 |

- **显示策略**：
  - 仅显示"一个月省的电"卡片
  - 场景文案大字号、橙色强调
  - 每个次数对应不同场景

## 8. 异常与边界处理

| 异常场景 | 处理方案 |
|----------|----------|
| 非支付宝环境 | 显示模拟数据，提示用户在手机设置中开启 NFC |
| JSAPI 调用失败 | 降级为手动引导，显示"设置 > 连接 > NFC"路径 |
| 用户拒绝跳转 | 保留当前页面，允许重新尝试 |
